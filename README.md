## 一、先把概念捋顺：IPLC 和 IEPL 到底是什么

很多人把这两个词混着用，其实它们在 OSI 模型里站的位置都不一样。

**IPLC（International Private Leased Circuit，国际私有专线）** 是老牌选手。它基于 SDH/DDN 传输技术，走的是传统的电路交换思路——发送端把数据封装成固定时隙的帧结构，通过预先建立的物理路径传输，接收端再重组。说白了，就是给你拉一条"专属铁轨"，从 A 地直通 B 地，别人上不来，你也不绕路。

**IEPL（International Ethernet Private Line，国际以太网专线）** 是后来者。它构建在 MSTP 设备平台上，基于 SDH 传输技术，但采用 GFP（Generic Framing Procedure）封装协议，把以太网帧映射到 SDH 的 VC 通道里。本质上，它是以太网接口的"增强版专线"，属于二层链路层服务。

一句话总结：**IPLC 是物理层的"硬隔离"，IEPL 是链路层的"软隔离但同等级别干净"**。两者都不走公网，都不与别的流量混杂，但实现路径不同。

## 二、IPLC和IEPL延迟差异的底层逻辑

延迟这事，说复杂也复杂，说简单也简单。把它拆开看，主要由四部分组成：

$$\text{总延迟} = \text{传播延迟} + \text{传输延迟} + \text{处理延迟} + \text{排队延迟}$$

其中传播延迟是物理决定的，光速摆在那儿，谁也改不了。每 1000 公里大约 5ms，这是天花板。

那么 IPLC 和 IEPL 的延迟差，主要来自后面三项：

### 1. 协议层级带来的处理开销

IPLC 走的是电路交换，时隙固定，数据进来直接走，没有路由表查询、没有 MAC 地址学习这些动作。处理延迟极小。

IEPL 虽然也是二层直连，但毕竟是以太网帧在跑，要经过 GFP 封装/解封装、MSTP 平台调度，理论上比纯物理电路多一层处理。但这一层开销在毫秒级以下，对绝大多数业务几乎无感。

### 2. 抖动（Jitter）表现

这才是两者真正拉开差距的地方。从行业实测数据看：

- **IPLC 延迟波动**：基本在 2~3ms 以内
- **IEPL 延迟波动**：通常在 5~8ms 之间

也就是说，平均延迟两者可能差不了几毫秒，但 IEPL 的抖动会比 IPLC 稍大一点。对于视频会议、跨境直播这种对稳定性敏感的场景，这个差距能感知到；对于普通网页访问、API 调用，几乎可以忽略。

### 3. 路由路径的差异

IPLC 是点对点的物理专线，路径最短最纯粹。IEPL 在某些实现里会引入二层设备做"多端支持"，理论上多一跳就多一点延迟。但优秀的 IEPL 商家会做点对点优化，把这个差距压到几乎测不出来。

> 这里要提醒一句：网上有文章说"IEPL 延迟一定比 IPLC 低"或"IPLC 一定比 IEPL 快"，这两种说法都不严谨。**真实延迟主要取决于物理距离和商家的工程能力，而不是协议名字本身**。

## 五、延迟之外的三个指标：抖动、丢包、稳定性

光看延迟数字会掉坑里。真正决定体验的，是延迟的"稳不稳"。跨境网络常见三大坑：

**晚高峰丢包**：晚上 8~11 点，国际出口带宽被挤爆，TCP 重传飙升，延迟从 30ms 跳到 200ms。这是公网跨境最常见的死穴。

**路由绕行**：数据包没走最短路径，绕了半个地球才到目的地。比如上海到日本，正常 25ms，绕道美国再回来就变 250ms。

**随机时延跳变**：链路本身没问题，但某个中间节点偶发性抖动，导致业务闪断。

专线之所以是专线，就是要把这三个坑填平。MKCloud 的做法是：物理层走专属海缆（IPLC）或二层直连（IEPL），骨干层接多重冗余路径，路由层用 BGP 自动切换。极端情况（比如海缆故障）下，能自动切到备用路径，最大限度保障业务连续性。

## 九、常见疑问：关于IPLC和IEPL延迟的几个高频问题

**Q1：IPLC 真的比 IEPL 延迟更低吗？**
不一定。两者平均延迟差距通常在几个毫秒内，主要差异在抖动——IPLC 抖动更小（2~3ms），IEPL 抖动稍大（5~8ms）。对延迟绝对值敏感选 IPLC，对成本和灵活性敏感选 IEPL，两者都不走公网，都比普通 VPN 强一大截。

**Q2：为什么我的沪日 IPLC 实测延迟比官网标的 25ms 高？**
官网标的是"端内延迟"（入口机房到出口机房的延迟），你的实际延迟还要加上"你本地到入口机房"这一段。比如你在北京，到上海电信入口可能要 20ms+，加上端内 25ms，实测就是 45ms 左右。这是物理距离决定的，谁也压不下去。

**Q3：晚高峰延迟会飙升吗？**
MKCloud 走的是专线+ BGP 动态切换，不依赖公网出口带宽，理论上晚高峰影响远小于普通跨境线路。但物理海缆层面的偶发波动无法完全避免，商家通过多重冗余路径做兜底。

**Q4：共享带宽和独享带宽，延迟有区别吗？**
延迟本身区别不大，差异在稳定性。共享带宽高峰期可能因争抢出现波动，独享带宽资源独占，带宽波动几乎为零。专业直播团队、大体量广告投放商建议选独享。

**Q5：上海电信入口比广州入口快吗？**
看你业务方向。做日本/美国市场，上海电信入口国际出口优先级更高，无跨省无跨网 QoS 限速；做香港/东南亚，广州入口物理距离更近，端内延迟更低。两个入口各有所长，没有绝对优劣。
